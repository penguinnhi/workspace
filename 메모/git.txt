-- 형상관리할 파일, 프로젝트를 github에 올리기
1. github에 올릴(공유할) 파일이 있는 폴더에서 git bash 실행
2. git init 명령어로 해당 폴더에 git 초기화 
	ㄴ 위의 명령어를 실행하면 숨김폴더로 .git이 생성됨
	이 폴더가 형상관리를 진행하는 모든 내용을 포함하고 있음 
	.git폴더를 삭제하면 git과 연결이 해제됨
3. master 브랜치를 main으로 변경
 git branch -M main
4. 폴더 안의 내용을 공유할 remote repository를 github 사이트에 생성
5. 만든 폴더와 remote repository 연결
 git remote add origin 원격 저장소 url 
6. remote repository에 올릴 파일을 스테이징
 git add 파일명
 모든 파일을 올리고 싶다면 git add .
7. 커밋, 푸시 진행 
 git commit -m "커밋메시지" ( commit 한다고 원격 저장소에 올라가지 않음 --> 프로젝트의 복원시점을 만들어줌 
 git push origin main ( 올릴 브랜치명
8. 파일 수정 및 추가, 삭제 시에 6,7번 반복
 * git add . 명령어 실행시 LF~ 경고가 뜨면 
 git config --global core.autocrlf true 명령어 실행 후 
 다시 git add . 실행

-- 깃허브에 올린 파일을 내려받기
1. 팀장의 프로젝트와 동일한 경로에서 git bash 실행 
2. 팀장의 원격저장소에 있는 파일을 내려받기
 git clone 원격저장소 url

* 팀원이 올린 파일 내려받기
 git pull origin main


-- 브랜치 관련 명령어 
1. 현재 브랜치의 목록 확인 : git branch (*:현재 브랜치)
2. 새로운 브랜치 생성 : git branch 브랜치명 
3. 브랜치 이동 : git checkout 이동할 브랜치명
4. 개인 브랜치에서 main 브랜치로 합치기 
	ㄴ main 브랜치로 이동 
	ㄴ git merge 병합할 브랜치명
